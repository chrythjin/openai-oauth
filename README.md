# openai-oauth

[NPM](https://www.npmjs.com/package/openai-oauth) | [GitHub](https://github.com/EvanZhouDev/openai-oauth) | [Legal](#legal)

Free OpenAI API access with your ChatGPT account. This monorepo provides a localhost proxy and a Vercel AI SDK provider powered by ChatGPT OAuth tokens.

---

## 🏛 Original Project Features

This is a community-maintained tool that leverages OpenAI's Codex CLI endpoint (`chatgpt.com/backend-api/codex/responses`) to provide API access via your ChatGPT account.

- **Monorepo Structure**:
  - `packages/openai-oauth`: CLI and localhost proxy.
  - `packages/openai-oauth-core`: Shared transport and auth logic.
  - `packages/openai-oauth-provider`: Vercel AI SDK provider.
- **Compatibility**: Supports `/v1/chat/completions`, streaming, and tool calls.
- **Auth**: Uses standard `auth.json` generated by `npx @openai/codex login`.
- **Stateless Proxy**: A simple bridge between OpenAI-compatible clients and the Codex backend.

## 🛠 Custom Workspace Additions (This Repository)

> [!IMPORTANT]
> The following features and operational tools are **custom enhancements** added to this specific repository for stable workstation operation and advanced token management.

### 1. Token Vault & Rotation System
This repo includes an advanced token management layer located in `.codex/`.
- **Vault System**: Stores multiple tokens in `~/.codex/vault/` for easy switching.
- **Automated Rotation**: Scripted logic to rotate `auth.json` across multiple account slots.
- **Interactive Launchers**:
  - `.\.codex\launchers\manage-tokens.bat`: (Windows) Full interactive menu for token/service management.
  - `./.codex/launchers/manage-tokens.command`: (macOS) Double-clickable script for token management on Mac.
  - `.\.codex\launchers\rotate-next-token.bat`: (Windows) One-click rotation to the next slot.

### 2. Windows Service Management
Configured as a persistent background service to ensure high availability.
- **Service Name**: `OpenAIOAuthProxy` (Managed via NSSM).
- **Control Interface**:
  ```bash
  bun run token status   # Check server health and active token slot
  bun run token rotate   # Rotate token and restart service
  bun run token restart  # Clean restart of the Windows service
  ```
- **Logging**: Integrated I/O redirection to `C:\Logs\OpenAIOAuthProxy\`.

### 3. Model Aliases for OMO/OpenCode
Use model-name aliases when OMO/OpenCode should display the selected reasoning effort in the model name.

- **Available Aliases**: All GPT-5.x models are exposed with five reasoning-effort suffixes (`minimal`, `low`, `medium`, `high`, `xhigh`) for a total of 6 variants per model:
  - **GPT-5.5**: `gpt-5.5`, `gpt-5.5-minimal`, `gpt-5.5-low`, `gpt-5.5-medium`, `gpt-5.5-high`, `gpt-5.5-xhigh`
  - **GPT-5.4**: `gpt-5.4`, `gpt-5.4-minimal`, `gpt-5.4-low`, `gpt-5.4-medium`, `gpt-5.4-high`, `gpt-5.4-xhigh`
  - **GPT-5.4-mini**: `gpt-5.4-mini`, `gpt-5.4-mini-minimal`, `gpt-5.4-mini-low`, `gpt-5.4-mini-medium`, `gpt-5.4-mini-high`, `gpt-5.4-mini-xhigh`
  - **GPT-5.2**: `gpt-5.2`, `gpt-5.2-minimal`, `gpt-5.2-low`, `gpt-5.2-medium`, `gpt-5.2-high`, `gpt-5.2-xhigh`
  - **Other**: `gpt-5.3-codex`, `codex-auto-review`
- **OMO/OpenCode Model Setting**: Configure the model as an alias ID, for example:
  ```json
  {
    "model": "openai-oauth/gpt-5.4-mini-xhigh"
  }
  ```
- **Upstream Mapping**: `gpt-5.4-mini-xhigh` is sent upstream as model `gpt-5.4-mini` with reasoning effort `xhigh`.
- **Do Not Rely on `variant`**: For this provider, the visible effort should come from the model ID alias, not a separate `variant` field.
- **Restart Required**: OpenCode restarts do not reload the local proxy. After rebuilding or changing the proxy, restart `OpenAIOAuthProxy` with administrator rights, for example `.\.codex\launchers\manage-tokens.bat restart`.
- **Verify Before Use**: Confirm the live proxy exposes aliases with `curl http://127.0.0.1:10531/v1/models` before expecting OMO/OpenCode to list them.

Apply aliases to OMO/OpenCode in this order:

1. Rebuild this package if the proxy code changed: `bun run build`.
2. Restart the Windows proxy service from an administrator terminal:
   ```powershell
   .\.codex\launchers\manage-tokens.bat restart
   ```
3. Confirm the live service exposes the alias model:
   ```powershell
   Invoke-WebRequest http://127.0.0.1:10531/v1/models -UseBasicParsing
   ```
   The response should include entries such as `gpt-5.4-mini-xhigh` or `gpt-5.5-high`.
4. Update the local OMO/OpenCode model setting to the alias ID, for example `openai-oauth/gpt-5.4-mini-xhigh`. Do not leave it at `openai-oauth/gpt-5.4-mini` if you expect `xhigh` to be visible.
5. Restart OMO/OpenCode after saving the setting.
