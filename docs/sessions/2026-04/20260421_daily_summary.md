# 2026-04-21 Daily Summary - Token Rotator Bug Analysis & Hardening

## 1. Token Rotator 프로세스 제어 버그 분석
- **현상:** `--rotate` 또는 `--use` 명령어 실행 시 스크립트가 아무런 출력 없이 비정상 종료되거나 토큰이 실제 프록시에 반영되지 않는 현상 발생.
- **원인 분석:**
    - `stopProxy()` 함수가 `taskkill /F /IM bun.exe` 명령을 실행하여, **프록시 프로세스뿐만 아니라 스크립트를 실행 중인 본인 프로세스까지 함께 종료**시키는 설계상 결함 발견.
    - Windows 환경에서 `execSync`를 이용한 `start /B` 실행이 비동기적으로 분리되지 못하거나 부모 프로세스 종료 시 함께 종료되는 불안정성 확인.
- **보고:** `20260421_token-rotator-bug-report.md`를 통해 문제 원인과 수정 방향(포트 기반 종료, `spawn` 방식 교체 등)을 상세히 공유.

## 2. Token Rotator 안정화 및 버그 수정 완료
- **주요 수정 사항:**
    - **프로세스 관리 개선:** `taskkill /IM bun.exe`를 제거하고, 포트 10531을 사용하는 PID를 정확히 타겟팅하여 종료하도록 수정.
    - **프로세스 시작 로직 교체:** `spawn()` + `detached: true` + `unref()` 방식을 사용하여 부모 프로세스 종료 후에도 프록시가 독립적으로 실행되도록 변경.
    - **PID 추적:** 실제 실행된 프록시의 PID를 `.codex/proxy.pid` 파일에 저장하여 추후 종료 명령 시 정확한 프로세스를 제어할 수 있도록 함.
    - **상태 검증 자동화:** 프록시 시작 후 `/health` 엔드포인트에 폴링(polling)을 수행하여, 실제 서버가 정상적으로 응답할 때만 '시작 성공'으로 간주하도록 강화.
- **결과:** 토큰 전환 후 프록시가 자동으로 안전하게 재시작되며, `--status` 명령을 통해 활성화된 계정 정보를 정확히 확인할 수 있게 됨.
